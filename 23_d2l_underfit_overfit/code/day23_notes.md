# Day23：模型选择、欠拟合和过拟合

**核心主题：**训练误差 vs 泛化误差 → 验证集 → K折交叉验证 → 欠拟合与过拟合 → 模型容量 → 多项式拟合实验

---

## 一、核心概念

### 1\.1 训练误差 vs 泛化误差

|误差类型|定义|计算数据|
|---|---|---|
|**训练误差**|模型在训练集上的误差|训练集|
|**泛化误差**|模型在"没见过的新数据"上的误差|测试集|

🎯 **我们真正关心的是泛化误差**——模型放到真实场景（森林里拍照识别树种）好不好用，看的不是它在训练过的图片上表现多好，而是遇到新图片时的表现。

### 1\.2 验证集

从训练数据中再切出一部分作为验证集（Validation Set），用于**调超参数和选择模型**，测试集只在最后用一次。

原始数据 → 训练集（训练参数） \+ 验证集（选模型/调超参） \+ 测试集（最终评估）

|数据集|用途|使用频率|
|---|---|---|
|训练集|更新参数（W, b）|每轮都用|
|验证集|调超参数、选模型结构|多次使用|
|测试集|最终评估泛化能力|**只用一次**|

⚠️ 不能拿测试集调参——否则测试集"泄露"进训练过程，泛化误差的估计就不可信了。

### 1\.3 K 折交叉验证

当数据量少时，把训练数据分成 K 份：

第1轮：用第 1 份验证，其余 K\-1 份训练

第2轮：用第 2 份验证，其余 K\-1 份训练

\.\.\.

第K轮：用第 K 份验证，其余 K\-1 份训练

最终误差 = K 次验证误差的平均值

常见选择：K = 5 或 K = 10。

## 二、欠拟合与过拟合

### 2\.1 三种情况

|欠拟合(Underfitting)|正常(Just Right)|过拟合(Overfitting)|
|---|---|---|
|模型太简单|模型复杂度刚好|模型太复杂|
|训练误差高|训练误差低|训练误差极低|
|泛化误差高|泛化误差低|泛化误差高|
|连训练集都学不好|训练集和新数据都行|死记硬背训练集|


### 2\.2 判断标准

|现象|诊断|
|---|---|
|训练误差很高|**欠拟合**——模型容量不够|
|训练误差低 \+ 测试误差高|**过拟合**——模型在背答案|
|训练误差低 \+ 测试误差低|✅ 刚好——泛化能力好|

### 2\.3 模型容量

模型容量 ≈ 模型拟合复杂函数的能力。容量越高，能拟合的模式越复杂，但也越容易过拟合。

低容量 ──────────── >高容量

线性模型     MLP(256\)    |   ResNet\-152      GPT\-4

(欠拟合风险高\)           |             (过拟合风险高\)

📐 **奥卡姆剃刀原则**：在效果相近的模型中，选最简单的那个。

## 三、数据生成（多项式拟合实验）

本实验是理解欠拟合/过拟合的**最经典 Demo**。

### 3\.1 真实函数

一个 3 次多项式（4 个系数）：

y = 5 \+ 1\.2*x \- 3\.4*x² \+ 5\.6*x³ \+ 噪声

在代码中表现为：

```python
max_degree = 20                    # 最多生成 20 个特征 x⁰, x¹, ..., x¹⁹
true_w = np.zeros(max_degree)      # 只有前 4 个不为 0
true_w[0:4] = [5, 1.2, -3.4, 5.6]  # 对应 x⁰, x¹, x², x³ 的系数
```

### 3\.2 多项式特征构造

```python
# 原始特征：200 个随机 x 值，shape=(200, 1)
features = np.random.normal(size=(200, 1))

# 生成 x⁰, x¹, x², ..., x¹⁹ 共 20 个特征
poly_features = np.power(features, [0, 1, 2, ..., 19])

# shape: (200, 20)
# 每列除以 i!（泰勒展开的归一化）
for i in range(20):
    poly_features[:, i] /= math.gamma(i + 1)  # gamma(i+1) = i!
```

🧠 为什么要除以 i ？ 因为 x³ 增长比 x² 快得多，除以阶乘能让各阶特征在数值上保持同一量级，梯度下降更稳定。

### 3\.3 数据划分

```python
n_train, n_test = 100, 100        # 各 100 个样本
# 前 100 个训练，后 100 个测试
```

## 四、实验设计

|实验|输入特征|特征数|模型参数数|真实函数阶数|
|---|---|---|---|---|
|正常|前 4 阶 (x⁰\~x³\)|4|4|3 次多项式 ✅|
|欠拟合|前 2 阶 (x⁰, x¹\)|2|2|只能用 1 次曲线 ❌|
|过拟合|全部 20 阶|20|20|自由度远多于真实需要 ❌|

### 4\.1 训练配置

```python
loss = nn.MSELoss(reduction='none')          # 均方误差
net = nn.Sequential(nn.Linear(特征数, 1, bias=False))  # 单层线性，无偏置
trainer = torch.optim.SGD(net.parameters(), lr=0.01)
```

为什么 bias=False？因为真实数据的偏置项已经作为 x⁰ 特征（系数 = 5\.0）包含在多项式里了。

## 五、实验结果解读

### 5\.1 正常拟合（4 阶特征，400 epoch）

训练集和测试集两条 loss 曲线一起下降，最终都收敛到很低的值。

打印学到的权重：

weight: [[ 5\.0   1\.2  \-3\.4   5\.6 \]\]   ← 几乎完美复原 true\_w！

✅ 现象：train loss ≈ test loss ≈ 0\.01（噪声方差），模型学到了真实规律。

### 5\.2 欠拟合（2 阶特征，400 epoch）

模型只有 x⁰ 和 x¹ 两个特征，只能拟合直线或抛物线中的一种（实际上是一次 \+ 常数项），无法表达 x² 和 x³ 的弯曲。

❌ 现象：train loss 和 test loss 都居高不下，两条曲线几乎重合但都很大——模型太简单，连训练集都学不好。

### 5\.3 过拟合（20 阶特征，1500 epoch）

模型有 20 个自由度，远超真实所需的 4 个。前几个 epoch 两条曲线一起降，但到了某个点：

训练 loss 继续下降（甚至趋近于 0）

测试 loss 反而上升了——两条曲线分叉

❌ 现象：train loss 很低（≈ 0），test loss 很高——模型把训练集的噪声也"背"下来了，失去泛化能力。

### 5\.4 三条 loss 曲线速记

欠拟合 (2阶\)：    train loss 高 ───  test loss 高      （两条一起高）

正常 (4阶\)：      train loss 低 ───  test loss 低      （两条一起低）

过拟合 (20阶\)：   train loss 极低 ─   test loss 高      （分叉！）

## 六、如何解决欠拟合和过拟合？

|问题|解决方案|
|---|---|
|欠拟合|增加模型容量：加层数、加神经元、换更强模型|
|过拟合|① 权重衰减（L2 正则化）② Dropout（丢弃法）③ 早停（Early Stopping）验证集 loss 不再下降就停④ 增加训练数据（数据增广）|

## 七、数据量与模型容量的关系

一个实用的经验法则：

模型参数量应该与训练数据量相匹配。数据越少，模型应该越简单。

少量数据 → 简单模型（防过拟合）

大量数据 → 复杂模型（充分利用数据）

## 八、关键代码模式

### 8\.1 多项式特征生成（NumPy 经典写法）

```python
# (200, 1) 的原始特征
features = np.random.normal(size=(n_train + n_test, 1))

# 一行生成 x⁰ 到 x¹⁹
poly_features = np.power(features, np.arange(max_degree).reshape(1, -1))

# np.arange(20).reshape(1, -1) → shape (1, 20)，即 [0,1,2,...,19]
# np.power(features, [0,1,...,19]) → x⁰, x¹, ..., x¹⁹

# 除以 i! 做归一化
for i in range(max_degree):
    poly_features[:, i] /= math.gamma(i + 1)
```

### 8\.2 同时绘制 train/test loss 的 Animator

```python
animator = Animator(xlabel='epoch', ylabel='loss', yscale='log',
xlim=[1, num_epochs], ylim=[1e-3, 1e2],
legend=['train', 'test'])
animator.add(epoch, (train_loss, test_loss))
```

yscale='log'：loss 跨多个数量级时对数坐标更清晰。
