# Day10：图像放缩与插值 \+ 图像翻转

## 一、图像放缩与插值 — `cv.resize()`

### 1\. 为什么需要图像放缩

|场景|说明|
|---|---|
|深度学习输入|模型要求固定尺寸（如 224×224），必须缩放|
|图像金字塔|多尺度检测，逐层缩放|
|显示适配|图片太大超出屏幕，需要缩小显示|
|速度优化|缩小后再处理，大幅减少计算量|
|数据增强|训练时随机缩放，提高模型泛化能力|

🌲 **林业场景**：无人机航拍图分辨率极高（4000×3000），直接处理太慢，通常先缩小到 640×480 再做分析。

### 2\. `cv.resize()` — 核心 API

```python
dst = cv.resize(image, (0,0), fx=0.75, fy=0.75, interpolation=cv.INTER_NEAREST)
```

#### 参数详解

|参数|值|含义|
|---|---|---|
|src|image|输入图像|
|dsize|(0,0\)|目标尺寸（宽, 高），设为 0 时不生效，由 fx/fy 决定|
|fx|0\.75|宽度缩放因子（0\.75 = 缩小到 75%）|
|fy|0\.75|高度缩放因子（0\.75 = 缩小到 75%）|
|interpolation|cv\.INTER\_NEAREST|插值方法|

### 3\. 两种指定尺寸的方式

#### 方式一：指定缩放因子（比例缩放）

```python
dst = cv.resize(image, (0,0), fx=0.75, fy=0.75, interpolation=cv.INTER_NEAREST)
```

dsize = (0,0\) → 忽略直接指定尺寸

新尺寸 = \(原宽 × fx, 原高 × fy\)

如原图 800×600，fx=fy=0\.75 → 新图 600×450

#### 方式二：指定目标尺寸（精确缩放）

```python
dst = cv.resize(image, (w//2, h//2), interpolation=cv.INTER_NEAREST)
```

dsize = \(w//2, h//2\) → 直接指定新尺寸

注意顺序是 \(宽度, 高度\)，不是 \(高度, 宽度\)

// 是整除，避免浮点数

|方式|适合场景|示例|
|---|---|---|
|fx，fy|按比例缩放，不关心具体像素数|缩放到原来的 50%|
|dsize|需要精确尺寸（如模型输入）|缩放到 224×224|

### 4\. 图像尺寸获取

```python
h, w, c = image.shape
```

|变量|含义|说明|
|---|---|---|
|h|高度（Height）|行数，shape[0\]|
|w|宽度（Width）|列数，shape[1\]|
|c|通道数（Channels）|shape[2\]，BGR=3，灰度图无此维度|

### 5\. 插值方法详解（重点！）

```python
cv.resize(..., interpolation=cv.INTER_NEAREST)
```

当缩放图像时，新像素的位置不一定正好落在原始像素上，需要用插值来估算新像素值。

#### 常用插值方法

|插值方法|OpenCV 常量|原理|效果|速度|
|---|---|---|---|---|
|最近邻|cv\.INTER\_NEAREST|取离新位置最近的原始像素值|马赛克感，边缘锯齿|⚡最快|
|双线性|cv\.INTER\_LINEAR|取周围 4 个像素的加权平均（默认）|平滑自然|中等|
|双三次|cv\.INTER\_CUBIC|取周围 16 个像素做三次插值|更平滑，细节好|较慢|
|Lanczos|cv\.INTER\_LANCZOS4|取周围 64 个像素做插值|最平滑，质量最高|🐢最慢|

#### 直观对比

```Plain Text
原图放大 3 倍：
最近邻:       [□][□][□]  每个像素直接复制，方块感明显
             [□][□][□]
             [□][□][□]
双线性:       [▥][▤][▦]  像素间平滑过渡，自然但稍模糊
             [▧][▨][▩]
             [▪][▫][▬]
双三次:       [▓][▒][░]  更精细的过渡，边缘更锐利
             [▒][▓][▒]
             [░][▒][▓]
```

#### 如何选择

|场景|推荐插值|
|---|---|
|缩小图像|cv\.INTER\_AREA（额外选项，抗锯齿）|
|放大图像，追求速度|cv\.INTER\_NEAREST|
|放大图像，通用场景|cv\.INTER\_LINEAR（默认，代码中不写也行）|
|放大图像，追求高质量|cv\.INTER\_CUBIC|

⚠️ 代码中用的是 INTER\_NEAREST，适合演示和学习，但实际项目放大图像通常用 INTER\_LINEAR 或 INTER\_CUBIC 获得更好的视觉效果。

## 二、图像翻转 — `cv.flip()`

### 1\. 翻转变换

沿着某条轴，把图像像照镜子一样反转。

🌲 **林业场景**：数据增强（左右翻转增加训练样本量）、校正无人机拍摄方向、镜像叶片图像增加样本多样性。

### 2\. `cv.flip()` — 核心 API

```python
dst = cv.flip(image, flipCode)
```

#### flipCode 参数

|flipCode|效果|说明|
|---|---|---|
|0|上下翻转|绕水平轴翻转，顶部变底部|
|1|左右翻转|绕垂直轴翻转，左侧变右侧（镜像）|
|\-1|对角线翻转|既上下又左右 = 旋转 180°|

#### 可视化

```Plain Text

  原图                        原图                        原图
```

### 3\. 翻转的实现原理

cv\.flip\(\) 本质上是重新排列像素索引：

```python
# flipCode=0 上下翻转 → 等同于
dst = image[h-1::-1, :, :]    # 行索引从末尾向开头读取
# flipCode=1 左右翻转 → 等同于
dst = image[:, w-1::-1, :]    # 列索引从末尾向开头读取
# flipCode=-1 对角线翻转 → 等同于
dst = image[h-1::-1, w-1::-1, :]   # 行列都反转
```

当然，实际中直接用 cv\.flip\(\)，效率远高于 NumPy 索引。

### 4\. 翻转 vs 旋转

|操作|效果|API|
|---|---|---|
|左右翻转|镜像|cv\.flip(image, 1\)|
|上下翻转|倒影|cv\.flip(image, 0\)|
|旋转 90°|转置 \+ 翻转|cv\.rotate(image, cv\.ROTATE\_90\_CLOCKWISE\)|
|任意角度|绕中心旋转|cv\.warpAffine\(\) \+ 旋转矩阵|

cv\.flip\(\) 只能做镜像翻转，任意角度旋转要用 cv\.warpAffine\(\)。

## 三、两个函数对比

|维度|resize\_demo\(\)|flip\_demo\(\)|

|功能|图像缩放 \+ 插值|图像翻转|
